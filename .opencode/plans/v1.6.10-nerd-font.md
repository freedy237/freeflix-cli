## Problème
- v1.6.9 a été pushé avec un code cassé (FlareSolverr), puis corrigé (fsschal cookie) mais PyPI a l'ancienne version
- Il manque l'auto-détection et l'installation de la Nerd Font

## Étapes

### 1. Bump → v1.6.10
- `pyproject.toml` : `version = "1.6.10"`
- `CHANGELOG.md` : entrée 1.6.10 avec les deux fixes
- Pas de nouveau tag v1.6.10

### 2. `setup_assistant.py` — `detect_nerd_font()`
```python
def detect_nerd_font() -> bool:
```
- **Linux** : `subprocess.run(["fc-list"], capture_output=True, text=True)` → cherche `(?i)CaskaydiaCove|Nerd Font`
- **macOS** : pareil via `fc-list` (installé via `brew` normalement)
- **Windows** : `os.path.exists` sur `%LOCALAPPDATA%\Microsoft\Windows\Fonts\CaskaydiaCoveNerdFont-*.ttf`
- Retourne `True`/`False`

### 3. `setup_assistant.py` — `install_nerd_font()`
```python
def install_nerd_font() -> bool:
```
Pattern `install_chafa()` :
- **Linux** :
  1. Télécharge `https://github.com/ryanoasis/nerd-fonts/releases/latest/download/CascadiaCode.zip`
  2. Extrait dans `~/.local/share/fonts/`
  3. `fc-cache -f ~/.local/share/fonts/`
- **macOS** : `brew install --quiet --cask font-caskaydia-cove-nerd-font`
- **Windows** :
  1. Télécharge le zip
  2. Extrait dans `%LOCALAPPDATA%\Microsoft\Windows\Fonts\`
  3. Enregistre dans le registre `HKCU:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Fonts`
- Interactive : demande Y/n avant d'installer
- Retourne `True`/`False`

### 4. `main.py` — Intégration dans les settings
Quand l'utilisateur choisit "nerd" (ligne 755) :
```python
if c == 1:  # nerd selected
    if not detect_nerd_font():
        if install_nerd_font():
            print_success("Nerd Font installed! Restart FreeFlix to apply.")
        else:
            print_warning("Install manually or keep emoji icons.")
    else:
        print_success("Nerd Font detected.")
    tracker.set_icon_style("nerd")
else:
    tracker.set_icon_style("emoji")
```
